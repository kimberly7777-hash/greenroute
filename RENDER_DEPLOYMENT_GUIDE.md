### **Step 3: Environment Variables**

Render will automatically read from `render.yaml`, but verify these are set:

#### **Required Environment Variables**:

```env
APP_NAME=AFIA-ORBIT
APP_ENV=production
APP_DEBUG=false
APP_URL=https://afia-orbit.onrender.com
APP_KEY=<auto-generated>

# Database (SQLite)
DB_CONNECTION=sqlite
DB_DATABASE=/var/www/html/database/database.sqlite

# Session
SESSION_DRIVER=database
SESSION_LIFETIME=120

# Cache
CACHE_STORE=database

# Queue
QUEUE_CONNECTION=database

# Logging
LOG_CHANNEL=stderr
LOG_LEVEL=info

# Mail (for production, update with real mail service)
MAIL_MAILER=log
MAIL_FROM_ADDRESS=hello@afia-orbit.com
MAIL_FROM_NAME=AFIA-ORBIT
```

### **What Happens During Deployment**:

1. **Build Phase**:
   ```
   ├── Pull Docker image (PHP 8.2 with Apache)
   ├── Install system dependencies
   ├── Install Composer dependencies
   ├── Install and build Node.js assets
   ├── Create SQLite database
   ├── Generate APP_KEY
   └── Set proper permissions
   ```

2. **Startup Phase** (via `start.sh`):
   ```
   ├── Generate APP_KEY if missing
   ├── Clear and cache configs
   ├── Run database migrations
   ├── Create storage symlink
   └── Start Apache server
   ```

## 🗄️ **Database Configuration**

### **Using SQLite** (Current Setup):

**Pros**:
- ✅ No external database needed
- ✅ Simple setup
- ✅ Works on free plan

**Cons**:
- ⚠️ Data resets on each deployment
- ⚠️ Not suitable for production with persistent data

### **Upgrade to PostgreSQL** (Recommended for Production):

1. **Add PostgreSQL Database on Render**:
   - Dashboard → New → PostgreSQL
   - Get connection details

2. **Update Environment Variables**:
   ```env
   DB_CONNECTION=pgsql
   DB_HOST=<your-postgres-host>
   DB_PORT=5432
   DB_DATABASE=<your-database-name>
   DB_USERNAME=<your-username>
   DB_PASSWORD=<your-password>
   ```

3. **Update `render.yaml`**:
   ```yaml
   - key: DB_CONNECTION
     value: pgsql
   - key: DB_HOST
     fromDatabase:
       name: afia-orbit-db
       property: host
   - key: DB_PORT
     fromDatabase:
       name: afia-orbit-db
       property: port
   - key: DB_DATABASE
     fromDatabase:
       name: afia-orbit-db
       property: database
   - key: DB_USERNAME
     fromDatabase:
       name: afia-orbit-db
       property: user
   - key: DB_PASSWORD
     fromDatabase:
       name: afia-orbit-db
       property: password
   ```

## 🐛 **Troubleshooting**

### **Issue 1: Deployment Fails**

**Check Build Logs**:
- Render Dashboard → Logs → Build Logs
- Look for errors in dependencies or build process

**Common Fixes**:
```bash
# Clear Composer cache
composer clear-cache

# Update dependencies
composer update --no-dev

# Rebuild assets
npm run build
```

### **Issue 2: 500 Server Error**

**Check Application Logs**:
- Render Dashboard → Logs → Application Logs

**Common Fixes**:
- Verify APP_KEY is generated
- Check database connection
- Verify file permissions
- Run migrations: `php artisan migrate --force`

### **Issue 3: Assets Not Loading**

**Fix**:
```bash
# In Render Shell
php artisan storage:link
php artisan config:clear
php artisan cache:clear
```

### **Issue 4: Database Resets**

**SQLite Issue**: 
- Data resets on each deploy with SQLite
- **Solution**: Upgrade to PostgreSQL (see above)
